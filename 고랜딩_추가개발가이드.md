# 고랜딩 추가 개발 가이드

## 1. 목적

이 문서는 고랜딩을 다음 단계로 개발할 때, 무엇을 먼저 보고 어떤 순서로 손대야 하는지 정리한 문서입니다.

이 프로젝트는 기능 자체보다도 아래 3개가 중요합니다.

- 속도
- 사용자 UX
- 분석 정확도

## 2. 현재 우선순위

### 1순위

- 홈 대시보드 체감 속도 추가 개선
- 랜딩 수정 후 저장 속도 개선
- 대시보드 목록 UX 안정화

### 2순위

- 공개 랜딩 이벤트 수집 최적화
- 분석 페이지의 시각화 품질 개선
- 관리자/회원관리 페이지 추가 분리 로딩

### 3순위

- 운영용 업로드 스토리지 정식 연결
- 메타 픽셀 추가 이벤트 지원
- 관리자 리포트 고도화

## 3. 현재 병목 지점

### 홈

현재 홈은 `/api/dashboard`를 통해 데이터를 불러옵니다.

현재 개선된 상태:

- 페이지 틀은 바로 뜸
- 캐시가 있으면 이전 데이터를 먼저 보여줌

추가 개선 후보:

- 숫자 카드와 랜딩 목록 API 분리
- 홈 랜딩 목록 첫 10개만 우선 표시
- 나머지는 더보기 또는 지연 로드

### 랜딩 수정 저장

현재 저장 후 홈으로 이동되도록 구성되어 있습니다.

하지만 저장이 느릴 수 있는 이유:

- 이미지/버튼/폼 필드 전체를 다시 저장하는 방식

추가 개선 후보:

- diff 기반 부분 업데이트
- 변경 없는 하위 리소스는 재삽입하지 않기
- 저장 중 로딩 표시 개선

### 공개 랜딩

현재 클릭/체류/폼 제출 데이터를 수집합니다.

추가 개선 후보:

- `sendBeacon` 활용
- 링크 클릭 시 이동 방해 최소화
- 체류 계산용 뷰포트 추적 최적화

## 4. 수정 시 특히 조심할 영역

### 4-1. 체류 분석

핵심 파일:

- `src/components/public-landing-view.tsx`
- `src/server/analytics-service.ts`
- `src/components/analysis-visuals.tsx`

이 영역은 사용자가 가장 민감하게 보는 부분입니다.

주의:

- 20구간 계산 기준이 바뀌면 수치 체감이 바로 달라짐
- 체류율, 스크롤 깊이, 오버레이 순위 기준을 섞지 말 것
- 문구 하나가 의미를 크게 바꿀 수 있음

### 4-2. 대시보드 목록

핵심 파일:

- `src/components/dashboard-landing-list.tsx`
- `src/app/globals.css`

주의:

- 버튼 추가 시 바로 줄바꿈/overflow가 생길 수 있음
- 버튼 간 폭, 토글 폭, metrics 영역 폭은 같이 봐야 함

### 4-3. 관리자/회원관리

핵심 파일:

- `src/app/admin/members/page.tsx`
- `src/app/admin/accounts/page.tsx`
- `src/components/approved-accounts-manager.tsx`
- `src/components/coupon-codes-manager.tsx`
- `src/components/signup-requests-manager.tsx`

주의:

- 회원관리와 운영관리 페이지를 다시 섞지 말 것
- 쿠폰/가입신청/승인계정은 회원관리 전용으로 유지

## 5. 추천 개발 순서

### 순서 1. 성능

- 홈 API 더 쪼개기
- 랜딩 저장 최적화
- 공개 랜딩 이벤트 전송 최적화

### 순서 2. 사용자 편의

- 대시보드 검색/정렬
- 랜딩 목록 페이지네이션
- 랜딩 복사본 이름 규칙 개선

### 순서 3. 관리자 편의

- 쿠폰 CSV 업로드
- 회원 만료 일괄 처리
- 가입 신청 검색/필터

### 순서 4. 분석 고도화

- 메타 픽셀 추가 이벤트 매핑
- CTA별 클릭 리포트
- 구간별 이탈 포인트 요약

## 6. 테스트 시 반드시 확인할 체크포인트

### 사용자 흐름

1. 회원가입
2. 로그인
3. 랜딩 생성
4. 발행
5. 공개 링크 방문
6. 분석 확인

### 관리자 흐름

1. 쿠폰 생성
2. 가입 신청 승인/반려
3. 승인 계정 수정/삭제
4. 관리자 첫 화면 진입

### 랜딩 관리 흐름

1. 수정
2. 저장
3. 홈 복귀
4. 복사
5. 삭제
6. 사용중/사용중지 토글

## 7. 문구/UX 관련 원칙

- 사용자에게 어려운 용어를 쓰지 말 것
- 가능한 한 짧고 직관적인 한글 문구 사용
- 설명 문구는 많을수록 좋은 게 아니라, 필요한 만큼만 둘 것
- 관리자 화면도 복잡한 표보다 한 줄형 관리가 우선

## 8. 운영 관련 원칙

- 운영 배포는 Vercel + Supabase 기준으로 유지
- 개발 기본 DB는 SQLite 유지 가능
- 이미지 업로드는 외부 스토리지 연결 전까지 URL 입력 우선
