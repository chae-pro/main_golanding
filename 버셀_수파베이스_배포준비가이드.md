# 고랜딩 버셀 + 수파베이스 배포 준비 가이드

## 현재 확정된 운영 기준

- 배포 플랫폼: `Vercel`
- 운영 DB: `Supabase Postgres`
- 관리자 이메일: `computerschool100@gmail.com`

## 이 조합을 쓰는 이유

- 고랜딩은 `Next.js SaaS` 구조이므로 Vercel 배포와 궁합이 좋음
- 운영 데이터는 앱 내부 파일이 아니라 외부 DB에 있어야 함
- 수파베이스는 PostgreSQL을 바로 제공하므로 운영 DB로 적합함

## 반드시 지켜야 할 기준

버셀 운영에서는 아래를 쓰면 안 됩니다.

- SQLite 파일을 운영 DB로 사용
- `public/uploads` 를 운영 업로드 저장소로 사용

운영 기준은 아래처럼 고정합니다.

```bash
GOLANDING_DB_PROVIDER=postgres
GOLANDING_STORAGE_PROVIDER=s3
```

## 다음에 당신이 해야 할 일

다음 순서는 `Supabase 프로젝트 생성` 입니다.

당신이 해야 하는 것은 아래입니다.

1. Supabase 계정 로그인
2. 새 프로젝트 생성
3. 프로젝트 비밀번호 저장
4. 프로젝트가 생성되면 `Connection string` 확인

## 이후 내가 이어서 할 일

당신이 Supabase 프로젝트를 만들고 나면, 그 다음은 내가 아래 순서로 안내합니다.

1. `DATABASE_URL` 형식 확인
2. Vercel 환경변수 입력값 정리
3. 운영용 이미지 저장소 선택
4. 첫 배포 순서 점검
