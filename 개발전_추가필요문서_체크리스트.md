## 1. 반드시 추가해두면 좋은 문서

### 1-1. 화면별 UX 기준서

목적:

- 버튼 위치, 문구, 간격, 정렬, 노출 순서를 다시 흔들리지 않게 고정

왜 필요한가:

- 이 프로젝트는 기능만큼이나 문구와 정렬 요구가 강함
- 대시보드, 분석, 회원관리 화면은 레이아웃 요구가 자주 바뀌었음

들어가야 할 내용:

- 홈 화면
- 대시보드 목록 행 구성
- 랜딩 생성/수정 화면
- 분석 화면
- 회원관리 화면
- 관리자 화면

권장 파일명:

- `고랜딩_화면별_UX기준서.md`

### 1-2. 분석 로직 기준서

목적:

- 체류, 클릭, 스크롤 계산의 기준을 명확히 고정

왜 필요한가:

- 지금까지 가장 민감했던 영역이 분석 수치와 오버레이 표현이었음
- 다음 개발자가 수식이나 기준을 바꾸면 바로 사용자 체감이 깨짐

들어가야 할 내용:

- 20구간 기준 정의
- 체류 퍼센트 산식
- 평균 체류 깊이 정의
- 클릭 수/버튼 클릭 수 구분
- 오버레이 색상/순위 기준
- 어떤 데이터는 보여주고 어떤 데이터는 숨기는지

권장 파일명:

- `고랜딩_분석로직기준서.md`

### 1-3. 성능 개선 기록 문서

목적:

- 느렸던 구간과 이미 최적화한 내용을 기록

왜 필요한가:

- 홈, 회원관리, 관리자, 저장 속도는 이미 여러 차례 최적화가 들어감
- 같은 실수를 반복하지 않기 위해 기록이 필요함

들어가야 할 내용:

- 느렸던 페이지
- 원인
- 적용한 개선
- 남은 병목
- 다음 최적화 후보

권장 파일명:

- `고랜딩_성능개선기록.md`

## 3. 개발 전에 준비해두면 좋은 비문서 자료

### 3-1. 테스트 계정 목록

- 일반 사용자 테스트 계정
- 관리자 테스트 계정
- 쿠폰 가입 테스트 계정

### 3-2. 테스트 쿠폰 목록

- 1일 쿠폰
- 7일 쿠폰
- 30일 쿠폰
- 정원 1명 쿠폰
- 정원 여러 명 쿠폰

### 3-3. 테스트 랜딩 샘플

- 버튼형 샘플
- DB 수집형 샘플
- HTML 삽입형 샘플

### 3-4. 테스트 시나리오

- 회원가입 -> 로그인
- 쿠폰가입 -> 로그인
- 랜딩 생성 -> 발행 -> 방문 -> 분석
- 랜딩 수정 -> 저장 -> 홈 복귀
- 랜딩 복사 -> 삭제
- 관리자 승인/반려/삭제
