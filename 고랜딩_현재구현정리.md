# 고랜딩 현재 구현 정리

## 1. 프로젝트 개요

고랜딩은 사용자가 랜딩페이지를 생성하거나 등록하고, 공개 링크를 배포한 뒤, 방문자의 행동 데이터를 분석하는 SaaS입니다.

현재 구현 범위는 크게 4개입니다.

- 사용자 로그인/회원가입/쿠폰 가입
- 랜딩 생성/수정/복사/삭제/발행 제어
- 공개 랜딩 페이지 제공
- 방문/클릭/폼 제출/체류 분석

## 2. 현재 동작하는 핵심 기능

### 2-1. 사용자 기능

- 승인 이메일 기반 로그인
- 회원가입 신청
- 쿠폰 기반 즉시 사용 가능 가입
- 내 랜딩 대시보드
- 랜딩 생성
- 랜딩 수정
- 랜딩 복사
- 랜딩 삭제
- 랜딩 발행/사용중지 토글
- 공개 링크 복사
- 분석 페이지 보기

### 2-2. 랜딩 유형

- 버튼형
- DB 수집형
- HTML 삽입형

### 2-3. 랜딩 생성/수정에서 지원하는 항목

- 제목
- 공개 슬러그
- 설명
- Meta Pixel ID
- 기본 색상
- 텍스트 색상
- 배경 색상
- 모서리 둥글기
- 이미지 URL 다중 등록
- 버튼 다중 등록
- DB 입력 필드 다중 등록
- 필드 순서 조정
- 버튼 순서 조정
- 이미지 순서 조정

## 3. 공개 랜딩 동작

공개 랜딩 경로:

- `/l/[publicSlug]`

현재 공개 랜딩에서 동작하는 것:

- 랜딩 이미지 표시
- 버튼형 CTA 표시
- DB 수집형 입력 폼 표시
- HTML 소스형 렌더링
- Meta Pixel `PageView` 삽입
- 방문 세션 생성
- 클릭 이벤트 수집
- 폼 제출 수집
- 스크롤/체류 계산용 데이터 수집

현재 공개 랜딩에서 의도적으로 단순화한 것:

- 버튼형 하단 설명 제거
- DB 수집형 상단 설명 제거
- 이미지 업로드 대신 이미지 URL 입력 우선 사용

## 4. 분석 기능

분석 페이지 경로:

- `/analysis/[landingId]`

현재 분석 페이지에서 보여주는 항목:

- 총 방문자
- 평균 머문 시간
- 평균 체류 깊이
- 총 클릭 수
- 버튼 클릭 수
- 폼 제출 수
- 구간별 체류그래프
- 랜딩 체류 보기
- 클릭 히트맵
- 데이터 리셋 버튼

현재 분석 기준:

- 랜딩 이미지를 기준으로 20구간 분할
- 구간별 체류 퍼센트 집계
- 오버레이는 구간 순위 기준으로 색상 강도 표시
- 오른쪽 요약 패널로 구간 퍼센트 표시

## 5. 관리자 기능

### 5-1. 회원관리 페이지

경로:

- `/admin/members`

현재 포함 항목:

- 쿠폰 관리
- 가입 신청 관리
- 승인 계정 관리

쿠폰 기능:

- 쿠폰 생성
- 쿠폰 수정
- 쿠폰 삭제
- 사용 기간(일) 설정
- 사용 가능 인원 설정
- 사용 인원/남은 인원 표시

가입 신청 관리:

- 승인
- 반려
- 삭제
- 반려 후 다시 승인 가능

승인 계정 관리:

- 이름 수정
- 기수 수정
- 상태 수정
- 만료일 수정
- 저장
- 삭제

### 5-2. 관리자 운영 페이지

경로:

- `/admin/accounts`

현재 포함 항목:

- 운영 관련 관리자 데이터
- 배포 준비 상태
- 기타 운영 요약 정보

## 9. 최근 구현상 중요한 결정

- 홈 대시보드는 서버 대기형이 아니라 API 기반 클라이언트 로드형
- 홈은 메모리/세션 캐시를 우선 사용
- 랜딩 저장 후 이동은 상세가 아니라 홈 대시보드
- 대시보드 목록은 한 줄 바 형태
- 공개 랜딩 이미지 업로드는 운영에서 비권장, 이미지 URL 입력 우선
- 회원관리와 운영 관리자 페이지 분리

## 10. 현재 남아 있는 주의사항

- 홈 로딩 체감은 여전히 추가 최적화 여지 있음
- 랜딩 저장 속도는 여전히 완전 빠르다고 보기 어려움
- 수정 저장은 내부적으로 중첩 리소스를 재저장하는 구조라 추가 최적화 가능
- 공개 랜딩의 클릭/체류 수집은 정확도와 성능을 계속 같이 봐야 함
- 관리자/회원관리 페이지는 더 세분화된 lazy load 여지가 있음
- 이미지 업로드를 운영에서 완전히 쓰려면 외부 스토리지 연결 필요

## 11. 다음 개발자가 가장 먼저 알아야 하는 것

- 이 프로젝트는 단순 랜딩 생성기가 아니라 분석 SaaS임
- 사용자 UX 요구가 매우 구체적이므로 문구/정렬/버튼 위치도 중요한 요구사항임
- 속도 문제는 기능 문제만큼 중요하게 다뤄야 함
- 홈, 회원관리, 관리자, 수정 저장은 성능 민감 구간임
- 분석의 핵심은 `20구간 체류 보기`이며, 이 부분은 앞으로도 가장 민감한 영역임
