# 고랜딩 사용방법

이 문서는 고랜딩을 실제로 사용할 때, 어떤 순서로 무엇을 해야 하는지 단계별로 설명하는 문서입니다.

대상은 2가지입니다.

- 일반 사용자
- 관리자

## 1. 일반 사용자 사용방법

### 1-1. 회원가입

경로:

- `/signup`

입력 항목:

- 이메일
- 이름
- 기수 또는 구분
- 메모
- 쿠폰

기본 규칙:

- 이메일과 이름은 필수
- 기수 또는 구분, 메모, 쿠폰은 선택
- 쿠폰이 유효하면 관리자 승인 없이 바로 사용 가능
- 쿠폰이 없으면 가입 신청 후 관리자가 승인해야 로그인 가능

### 1-2. 로그인

경로:

- `/login`

방법:

1. 승인된 이메일 입력
2. 로그인 버튼 클릭
3. 로그인 성공 후 홈 대시보드로 이동

## 2. 홈 대시보드 사용방법

경로:

- `/`

홈에서 할 수 있는 것:

- 랜딩 만들기
- 생성된 랜딩페이지 목록 확인
- 링크복사
- 수정
- 복사
- 삭제
- 분석
- 사용 중 / 사용중지 전환

### 2-1. 랜딩 만들기

홈 상단의 `랜딩 만들기` 버튼 클릭

이동 경로:

- `/landings/new`

### 2-2. 생성된 랜딩페이지 목록 보기

홈 하단의 `생성된 랜딩페이지` 영역에서 확인

한 줄마다 보이는 정보:

- 랜딩 제목
- 생성일
- 랜딩 형식
- 방문자 수
- 클릭 수
- 링크복사 / 발행전
- 수정
- 복사
- 삭제
- 분석
- 사용 중 / 사용중지

### 2-3. 링크복사

조건:

- 랜딩이 발행 상태여야 함

방법:

1. `링크복사` 버튼 클릭
2. 공개 랜딩 URL이 클립보드에 복사됨

### 2-4. 수정

방법:

1. `수정` 버튼 클릭
2. 수정 화면으로 이동
3. 내용 수정
4. 맨 아래 `랜딩 저장` 클릭
5. 저장 후 홈 대시보드로 이동

### 2-5. 복사

방법:

1. `복사` 버튼 클릭
2. 현재 랜딩 설정이 복사된 새 랜딩이 생성됨

복사 시 포함되는 것:

- 제목
- 랜딩 형식
- 이미지
- 버튼
- DB 입력 항목
- 색상 설정
- HTML 소스
- Meta Pixel ID

복사 시 포함되지 않는 것:

- 방문 데이터
- 클릭 데이터
- 체류 데이터
- 폼 제출 데이터

### 2-6. 삭제

방법:

1. `삭제` 버튼 클릭
2. 확인 팝업 확인
3. 삭제 완료

주의:

- 삭제하면 랜딩과 관련 분석 데이터가 함께 삭제됨

### 2-7. 사용 중 / 사용중지

방법:

1. 목록 오른쪽 토글 클릭
2. 상태가 변경됨

의미:

- 사용 중: 공개 링크 접속 가능
- 사용중지: 공개 링크 접속 시 공사중 화면 표시

## 3. 랜딩 생성/수정 사용방법

경로:

- 신규 생성: `/landings/new`
- 수정: `/landings/[landingId]/edit`

### 3-1. 공통 입력 항목

- 랜딩 유형
- 제목
- 공개 슬러그
- 설명
- Meta Pixel ID
- 기본 색상
- 텍스트 색상
- 배경 색상
- 모서리 둥글기

### 3-2. 랜딩 유형

선택 가능 유형:

- 버튼형
- DB 수집형
- HTML 삽입형

### 3-3. 이미지 등록

현재 권장 방식:

- 이미지 URL 입력

여러 장 등록 가능:

- 이미지 추가
- 순서 변경
- 삭제 가능

### 3-4. 버튼형

설정 가능 항목:

- 버튼 문구
- 이동 링크
- 버튼 폭 비율
- 버튼 순서 조정

### 3-5. DB 수집형

설정 가능 항목:

- 이름
- 이메일
- 전화번호
- 주소
- 메모1
- 메모2
- 메모3

각 항목에서 설정 가능한 것:

- 항목 유형
- 필수 여부
- 질문/라벨
- 안내 문구
- 순서 조정

### 3-6. HTML 삽입형

설정 가능 항목:

- HTML 소스 직접 입력

### 3-7. Meta Pixel ID

입력 위치:

- 랜딩 생성/수정 화면의 `Meta Pixel ID`

규칙:

- 숫자만 입력
- 입력하면 공개 랜딩 페이지에서 Meta Pixel `PageView` 실행

### 3-8. 저장

방법:

1. 화면 맨 아래 `랜딩 저장`
2. 저장 완료 후 홈 대시보드로 이동

## 4. 공개 랜딩 사용방법

경로:

- `/l/[publicSlug]`

현재 공개 랜딩에서 보이는 것:

- 등록한 랜딩 이미지
- 버튼형 CTA
- DB 수집형 입력 폼
- HTML 삽입형 내용

### 4-1. 버튼형 공개 랜딩

- 버튼만 보임
- 설명 문구는 없음

### 4-2. DB 수집형 공개 랜딩

- 입력칸과 제출 버튼만 보임
- 설명 문구는 없음

### 4-3. 사용중지 상태

- 공개 링크 접속 시 공사중 화면 표시

## 5. 분석 페이지 사용방법

경로:

- `/analysis/[landingId]`

보이는 항목:

- 총 방문자
- 평균 머문 시간
- 평균 체류 깊이
- 총 클릭 수
- 버튼 클릭 수
- 폼 제출 수
- 구간별 체류그래프
- 랜딩 체류 보기
- 클릭 히트맵

### 5-1. 데이터 리셋

방법:

1. `기존 데이터 리셋` 클릭
2. 확인 팝업 확인
3. 분석 데이터 초기화

주의:

- 이전 방문/체류/클릭 데이터가 삭제됨

## 6. DB 제출 내역 보기

경로:

- 랜딩 상세 화면
- 또는 `DB확인하기`

할 수 있는 것:

- 제출된 DB 확인
- CSV 다운로드

## 7. 관리자 사용방법

### 7-1. 회원관리 페이지

경로:

- `/admin/members`

관리 가능 항목:

- 쿠폰 관리
- 가입 신청 관리
- 승인 계정 관리

### 7-2. 쿠폰 관리

할 수 있는 것:

- 쿠폰 생성
- 쿠폰 수정
- 쿠폰 삭제

입력 항목:

- 쿠폰번호
- 사용 기간(일)
- 사용 가능 인원

동작:

- 회원가입 시 쿠폰 입력
- 유효 쿠폰이면 바로 사용 가능
- 사용 가능 인원 소진 시 더 이상 사용 불가

### 7-3. 가입 신청 관리

할 수 있는 것:

- 승인
- 반려
- 삭제

반려 후:

- 다시 승인 가능

### 7-4. 승인 계정 관리

할 수 있는 것:

- 이름 수정
- 기수 수정
- 상태 수정
- 만료일 수정
- 저장
- 삭제

## 8. 운영 관리자 페이지

경로:

- `/admin/accounts`

용도:

- 회원관리 외 운영 관련 데이터 확인
- 배포 준비 상태 확인

## 9. 추천 사용 순서

일반 사용자 기준 추천 순서:

1. 회원가입 또는 로그인
2. 랜딩 만들기
3. 이미지 URL 입력
4. 버튼 또는 DB 항목 설정
5. 랜딩 저장
6. 사용 중으로 전환
7. 링크복사
8. 외부에 배포
9. 분석 페이지에서 데이터 확인

관리자 기준 추천 순서:

1. 회원관리 진입
2. 쿠폰 생성
3. 가입 신청 확인
4. 승인 계정 관리
5. 필요 시 운영 관리자 페이지 확인

## 10. 주의사항

- 이미지 업로드보다 이미지 URL 입력이 현재 더 안정적
- 공개 링크는 `사용 중` 상태에서만 열림
- 삭제는 복구가 어려우므로 주의
- 분석 수치는 체류와 클릭 수집이 실제 반영된 이후 확인해야 함
